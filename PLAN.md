# Хакатон: AI CEO / Founder Strategist (AI-співробітник)

> План адаптирован под **реальный скаффолд этого репозитория**: Node 20 + TypeScript (ESM) + Express + `@anthropic-ai/sdk`, деплой на Railway (`railway.json` уже настроен). Ядро агента ложится на `src/agent.ts` / `src/index.ts`. Стек НЕ Next.js/Prisma — мы расширяем существующий бэкенд и добавляем лёгкий фронт.

## Context

Хакатон: построить «AI-співробітника» — агента, который **полностью** закрывает конкретную рабочую задачу (вход → готовый результат). Команда 3-4 чел., 1-2 дня. Приоритет жюри: **бизнес-ценность + чистое демо**.

**Концепция:** AI CEO / Founder Strategist.
**Вход:** бизнес-идея (свободный текст, опц. регион + бюджет).
**Выход:** инвесторского качества бизнес-план — sizing рынка (TAM/SAM/SOM с источниками), скан реальных конкурентов, Business Model Canvas, GTM / план 90 дней, юнит-экономика с редактируемыми допущениями, реестр рисков, скелет питч-дека + экспорт в PDF.

**Главный риск и как бьём:** концепция легко читается как «обёртка над ChatGPT». Антидот — 4 столпа: **grounding** (реальный веб-ресёрч с кликабельными источниками), **transparency** (видимый task-tree, как агент работает по шагам), **interactivity** (редактируемые допущения, мгновенно пересчитывающие юнит-экономику), **polished deliverable** (экспорт PDF/дек). Каждая фича обязана усиливать один из столпов — иначе режем.

Цель: за ~90 секунд из одной фразы — защищаемый перед инвестором бизнес-план. Работа, за которую консультант берёт $5k и 1-2 недели.

**Уточнённый результат (validation-скил):** не просто план, а **скоринговый отчёт валидации** — оценка идеи по 4 категориям (×25 → /100) + список gaps/вопросов для CustDev перед питчем. Баллы ставит **Multi-LLM панель** (Скептик-инвестор / Адвокат / Нейтральный аналитик), синтезатор усредняет и помечает расхождения >5 баллов как ключевые риски.

---

## Слой валидации + Multi-LLM панель

Поверх ресёрч-пайплайна (он собирает grounded-факты) работает скоринговый слой на базе **Lean Canvas** + критериев хакатона.

**4 категории (каждая из 25 → итог /100):**

| Категория | Что проверяем | Данные из шагов |
|---|---|---|
| Problem × Market | Болит ли? Кто платит и сколько? TAM/SAM/SOM, ICP, CustDev-подтверждение | brief + market |
| Solution × Differentiation | Почему это и почему сейчас? Топ-3 конкурента, unfair advantage | competitors |
| Business Model × Unit Economics | Сходится ли математика? Монетизация, LTV/CAC, путь к breakeven | canvas + unitEconomics |
| GTM × First Traction | Как берём первых клиентов? Каналы, первый сегмент, early signals | gtm |

**Multi-LLM панель (один LLM, разные системные промпты):**
- **Скептик-инвестор** — ищет дыры, задаёт жёсткие вопросы.
- **Адвокат** — strongest case за идею.
- **Нейтральный аналитик** — взвешенно, без bias.

Каждый прогоняет все 4 категории и ставит баллы. **Синтезатор** усредняет → итог /100 + список спорных точек (расхождение >5 баллов между персонами = главный риск и вопрос для CustDev). Это сильнейший антидот «обёртке»: видимое многоперспективное рассуждение и честные разногласия вместо одного монолога.

## Стек (адаптировано под репозиторий)

**Оставляем существующий бэкенд, расширяем его, добавляем лёгкий фронт.**

- **Backend:** текущий **Express + TypeScript (ESM) + `@anthropic-ai/sdk`** (`src/index.ts`, `src/agent.ts`). `npm run dev` (tsx watch) / `npm run start`. Деплой Railway уже сконфигурен (`railway.json`, healthcheck `/health`).
  - **Бамп SDK:** проверить, что версия `@anthropic-ai/sdk` поддерживает серверный `web_search` tool; при необходимости обновить (`^0.39.0` → актуальная).
- **LLM:** Claude API через SDK, с **prompt caching** общего контекста (структурированная идея + накопленный ресёрч) между шагами.
- **Grounding:** **нативный `web_search` tool Anthropic** — серверный, **цитаты возвращаются встроенно** (`cited_text`/`title`/`url`), не стоят токенов. Убирает интеграцию Tavily/Serper. Самое выгодное решение.
- **Персист (для реплея golden-прогона):** без Postgres — пишем `Run` как JSON на диск (`data/runs/<id>.json`) либо SQLite (`better-sqlite3`). Достаточно для хакатона и страховки демо.
- **Frontend (новый):** лёгкий **Vite + React** SPA, билдится в статику и **раздаётся тем же Express** (`app.use(express.static(...))`). Альтернатива — один HTML+JS файл. НЕ заводим отдельный Next.js/tRPC/Prisma — не окупается за 1-2 дня поверх готового бэкенда.
- **НЕ делаем в MVP:** auth, очереди/воркеры, реальная БД. Демо без логин-стены.

## Архитектура: оркестрированный DAG из 8 типизированных шагов

`src/agent.ts` из одиночного `runAgent` превращается в **оркестратор пайплайна** (предсказуемость для демо, типизированный JSON-выход на каждом шаге, параллель независимых).

| # | Шаг | Модель | web_search | Выход (типизированный JSON) |
|---|-----|--------|-----------|------------------------------|
| 1 | Структурирование идеи | Sonnet 4.6 | — | brief: проблема, клиент, value prop, гео, бюджет, research-вопросы |
| 2 | Sizing рынка TAM/SAM/SOM | **Opus 4.8** | ✅ | цифры + метод + каждая цифра ↔ citationId |
| 3 | Скан конкурентов | Sonnet 4.6 | ✅ | 4-8 конкурентов: позиционирование, цены, фандинг, URL, цитата |
| 4 | Business Model Canvas | Sonnet 4.6 | — | 9 блоков |
| 5 | GTM / 30-60-90 | Sonnet 4.6 | лёгкий | каналы, план, гипотезы |
| 6 | Юнит-экономика | **Opus 4.8** | лёгкий | объект `assumptions` + расчёт (см. ниже) |
| 7 | Реестр рисков | Sonnet 4.6 | — | риски: вероятность/импакт/митигация |
| 8 | Синтез → питч/one-pager | **Opus 4.8** | — | outline дека, executive summary, нарратив |

- **Модели:** Opus там, где качество рассуждения видно (sizing, юнит-экономика, синтез). Sonnet на извлечении/форматировании. Haiku 4.5 — утилиты (валидация «это вообще бизнес-идея?», заголовок).
- **Жёсткое правило (промпт + схема):** каждая количественная цифра несёт `citationId` из источников шага. Нет источника → `estimated: true` + явное допущение, **никогда не выдумывать цитату**. Флаг `estimated` — сигнал честности на демо.
- **Транспарентность (SSE):** новый эндпоинт `POST /agent/stream` (рядом с текущим `POST /agent`) эмитит события `step_started` → `tool_call(web_search, query=...)` → `source_found(title,url)` → `step_completed`. UI рисует **живой research-timeline** с реальными запросами и source-чипами. Все события пишем в `Run` → завершённый прогон реплеится мгновенно.

## Юнит-экономика — центр «это реальный инструмент»

Архитектура: **assumptions in → детерминированные формулы → outputs**. Формулы в **shared TypeScript (`src/shared/unitEconomics.ts`), НЕ в LLM**.

- Шаг 6 (Opus) выдаёт **только** объект `assumptions`, каждое поле `{ value, unit, rationale, citationId | estimated:true }` (ARPU/цена, gross margin %, CAC, churn, conversion, fixed cost, объём воронки…).
- Чистая функция `computeUnitEconomics(assumptions)` считает LTV (`ARPU × margin / churn`), LTV:CAC, payback, contribution margin, burn, **break-even**. Формулы видимы в тултипе.
- **Интерактивность:** каждое допущение — редактируемый input/слайдер; on-change клиент пересчитывает **локально, мгновенно, без LLM**. Главный «вау»: тянешь CAC — payback и break-even двигаются вживую. Чат-бот так не умеет.
- Sanity-bounds: SOM ≤ SAM; предупреждение при неправдоподобном LTV:CAC (читается как «агент самопроверяется»).
- Stretch: кнопка «re-evaluate with AI» — отправить отредактированные допущения в Opus на параграф sanity-check.

## Deliverable

- **In-app отчёт:** один скроллируемый sectioned-репорт: Executive Summary → Market (TAM/SAM/SOM funnel) → Competitors (таблица) → Canvas (сетка 9) → GTM 30-60-90 → Unit Economics (интерактив + чарты) → Risk matrix → Pitch outline. Inline-сноски цитат. Чарты — Recharts.
- **Экспорт — один основной, безупречный: PDF** (server-side, HTML-шаблон → PDF, напр. через Puppeteer/`@react-pdf`). Обязательно **Sources appendix** (URL + accessedAt) — артефакт-доказательство grounding.
- **Stretch:** PPTX-дек маппингом из шага 8.

## Разбивка по команде (3-4 чел., 2 дня)

**Общий kickoff (~1.5ч, все):** заморозить TypeScript-типы всех 8 шагов + `Assumptions` + `Citation` (`src/shared/types.ts`). **Типы — контракт**, после заморозки 4 потока параллелятся. Поднять `npm run dev`, проверить Anthropic-ключ, hello-world деплой на Railway.

- **Workstream A — Пайплайн агента + grounding** (сильнейший backend): D1 оркестратор в `src/agent/orchestrator.ts` + шаги 1-3 (citations, prompt caching). D2 шаги 4,5,7,8 + роутинг моделей + тюнинг промптов на дисциплину grounding.
- **Workstream B — Стриминг + персист + движок юнит-экономики** (backend/fullstack): D1 SSE-эндпоинт `/agent/stream` + схема событий, `Run` персист/реплей (JSON/SQLite), чистый `computeUnitEconomics` + тесты. D2 связка шага 6 → движок, опц. «re-evaluate».
- **Workstream C — Report UI + интерактив** (frontend): D1 Vite+React shell (раздаётся Express), форма ввода, live research-timeline (SSE), каркас секций. D2 все секции, чарты, сноски, **интерактивная панель юнит-экономики (слайдеры → мгновенный пересчёт)**, полировка.
- **Workstream D — Экспорт + демо + деплой** (4-й; при 3 чел. делят A-lead и C-lead на D2): D1 PDF-пайплайн, hardening Railway-деплоя. D2 PPTX-stretch, **2-3 golden-прогона** (пред-прогнанные, реплеятся мгновенно), репетиция, offline-fallback.

**MVP (обязан демо):** форма → live-timeline с реальными запросами+цитатами (шаги 2-3) → полный отчёт (8 секций) → интерактивная юнит-экономика с мгновенным пересчётом → экспорт PDF → один отполированный golden-прогон.

**Stretch (если зелено):** PPTX-дек, «re-evaluate with AI», deep-dive по странице цен конкурента, пресеты регионов, save/share-ссылка.

## Killer demo script (2.5 мин)

1. **Пре-фрейм (10с):** «Market-entry анализ у консультанта — $5k и 2 недели. Смотрите, как наш AI-сотрудник делает это за 90 секунд — и проверяйте источники по ходу».
2. **Вход (15с):** одна выверенная B2B-идея с проверяемыми данными. Пример: *«Подписка на охлаждённый ветеринарно-сформулированный свежий корм для собак с доставкой в Польше»* (реальные конкуренты Butternut Box / Lily's Kitchen, находимый рынок, понятные CAC/AOV/churn). Избегать «приложение для X» и гигантских идей.
3. **Прогон (90с):** агент печатает свой план-декомпозицию → live-поиск с видимыми tool-calls и кликабельными citation-чипами → таблица конкурентов наполняется в реальном времени → TAM/SAM/SOM funnel со сносками.
4. **WOW #1 (20с):** панель юнит-экономики. «Судья считает CAC оптимистичным — ставим 70 вместо 40». Меняем → LTV:CAC, payback, SOM-выручка и риск-флаг пересчитываются вживую.
5. **WOW #2 (15с):** клик **Export** → скачивается отформатированный PDF/дек с источниками-appendix. Заканчиваем на открытом деке.
6. **Закрытие (5с):** «Фундируемый бизнес-план с цитированными источниками и живой финмоделью — за 90 секунд».

**Правило демо:** нарративом подаём **доказательства** («вот откуда цифра»), а не прозу. ~90% демо — детерминированный реплей выверенного прогона, поданный как live; настоящую генерацию — на Q&A.

## Риски и митигации

1. **Live web-search падает на сцене** (главный) → демо из реплея сохранённого golden-Run (offline), live-генерация только в Q&A. Per-step таймауты + cached fallback. Spend-cap + второй API-ключ.
2. **Галлюцинированные цифры/фейк-цитаты** → schema-enforce `citationId`, серверная валидация что id есть в источниках шага, честный `estimated:true`. Opus на числовых шагах.
3. **Долгий агент рвёт стриминг/таймаут** → персистентный Express-сервер Railway (нет serverless-лимита), SSE с heartbeat, персист каждого события для реплея, per-step бюджеты токенов/времени.
4. **Стоимость/латентность** → prompt caching, Sonnet/Haiku на невидимых шагах, cap web_search per step, параллель независимых шагов.
5. **Математика расходится с прозой LLM** → математика в тестируемом чистом TS, LLM даёт только допущения. Один источник истины на каждую цифру.
6. **Schema churn стопорит параллель** → заморозка типов на kickoff (`src/shared/types.ts`), изменение схемы = тим-синк.
7. **Экспорт съедает D2** → PDF-from-HTML первым (надёжнее), PPTX только stretch, экспорт из персистнутого Run (развязан с live-агентом).

## Verification (end-to-end)

1. **Юнит-тесты движка:** `computeUnitEconomics` на известных входах даёт правильные LTV/payback/break-even; sanity-bounds (SOM ≤ SAM) срабатывают.
2. **Прогон пайплайна:** `curl` на `/agent/stream` с демо-идеей → все 8 шагов завершаются, каждый количественный факт несёт валидный `citationId` или `estimated:true`, события стримятся.
3. **Реплей:** перезагрузить сохранённый `Run` → timeline и отчёт восстанавливаются мгновенно без вызовов LLM.
4. **Интерактив:** изменить CAC в UI → LTV:CAC/payback/break-even/чарты пересчитываются локально без сети.
5. **Экспорт:** сгенерировать PDF → открывается, содержит все секции + Sources appendix с URL и accessedAt.
6. **Offline-тест демо:** прогнать весь сценарий в airplane mode накануне.
