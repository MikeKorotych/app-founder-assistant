# hahaton-2026 — инструкции для агентов

Проект: **AI CEO / Founder Strategist** — агент, превращающий бизнес-идею в скоринговый
отчёт валидации (4 категории ×25 → /100 + gaps) с Multi-LLM панелью. Полный план — [PLAN.md](PLAN.md).

## Живой план: обновлять при каждом push (ОБЯЗАТЕЛЬНО)

`plan.html` — живая витрина статуса проекта. **Перед каждым `git push`** обнови в нём
JSON-острова (только их, не верстку/CSS):

1. **`#data-progress`** — добавь запись **в начало** массива о том, что сделал:
   ```json
   { "date":"YYYY-MM-DD", "author":"<имя>", "workstream":"A|B|C|D",
     "summary":"<что сделано одной строкой>", "commit":"<short-sha>" }
   ```
2. **`#data-workstreams`** — если статус твоего воркстрима изменился, обнови `status`
   (`todo` → `in-progress` → `review` → `done`) и `owner`.

Дату бери из системного контекста сессии. `commit` можно проставить после коммита
(`git rev-parse --short HEAD`) отдельной правкой, либо оставить `"—"`.

## Командная синхронизация

- Перед работой: `git pull --rebase origin main`.
- Коммиты мелкие и частые, push сразу.
- `src/shared/types.ts` — **замороженный контракт**. Менять только синком со всей командой.
- Воркстримы пишут в разные файлы (A: `src/agent/*`, B: `src/store/*` + SSE, C: `web/*`),
  чтобы не конфликтовать. Не трогай чужие файлы без необходимости.

## Стек

Node 20 + TypeScript (ESM) + Express + `@anthropic-ai/sdk`, деплой Railway. ESM-импорты — с расширением `.js`.
`npm run dev` / `npm run typecheck` / `npm run start`.
